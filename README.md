# 水果打包加工 Demo

一个手机版最小可运行闭环：录入原料果 A/B、成品包数、次果和损耗，计算库存变化与成本分摊。

## 1. 怎么启动、怎么跑、怎么验证

### 启动

方式一：直接用浏览器打开 `index.html`。

### 怎么跑

1. 进入录入页。
2. 可以直接使用默认数据，也可以点击“示例”恢复示例数据。
3. 点击“生成结果”。
4. 如果数据合法，会进入结果页；如果数据不合法，会停留在录入页并提示错误。

### 最短验证

- 默认数据点击“生成结果”：应进入结果页。
- 示例数据结果应包含：原料消耗 `120 斤`、成品产出 `108 斤`、次果 `8 斤`、损耗 `4 斤`、守恒差异 `0 斤`、原料总成本 `280 元`。
- 把“损耗”改成 `0` 后点击“生成结果”：应停留在录入页，并提示投入产出不守恒。
- 运行语法检查：

```powershell
node --check app.js
```

## 2. 设计说明

### 为什么这么设计

这个题目的重点是可运行、可演示的最小闭环，不是完整业务系统。所以当前实现选择纯前端静态页面：

- 不依赖后端、数据库、构建工具，交付和打开成本低。
- 业务链路集中在一次加工批次：录入 -> 校验 -> 计算 -> 展示。
- 先把库存守恒、次果、损耗、成本结转讲清楚，避免堆功能。

### 核心结构

```text
index.html   页面结构：录入页、结果页
styles.css   手机优先样式
app.js       表单读取、校验、库存计算、成本计算、页面切换
README.md    交付说明
```

### 核心规则

- 原料果 A 成本价：`2 元/斤`
- 原料果 B 成本价：`3 元/斤`
- 成品规格限制为：`5 斤装`、`8 斤装`、`10 斤装`
- 成品重量 = 规格斤数 × 包数
- 产出重量 = 成品重量 + 次果 + 损耗
- 守恒差异 = 原料投入 - 产出重量
- 成本按成品、次果、损耗的重量占比分摊并分别列示

## 4. 风险与限制

### 已知风险

- 成本分摊规则是当前实现假设：按成品、次果、损耗重量占比分摊。题目没有给唯一成本处理方式。
- 不守恒数据被设置为阻断错误。这样更稳，但如果业务允许暂存异常加工单，需要后续调整。
- 当前没有自动化测试，主要依赖手动验证和语法检查。

### 当前仍可接受的原因

- 目标是最小可运行 demo，不是完整生产系统。
- 已覆盖核心闭环：原料投入、成品产出、次果、损耗、库存守恒、成本结转。
- 关键不合理输入会被阻断，避免展示误导性结果。
- 所有数据只在浏览器本地运行，不涉及上传、权限和数据安全风险。

### 暂不包含

- 后端、数据库、登录权限。
- 多批次历史、导出、审批流。
- 销售收入、利润、售价。
- 原料 A/B 到具体成品规格的追踪关系。
- 自定义成品规格。

## 5. 几分钟演示顺序

1. 打开 `index.html` 或 `http://127.0.0.1:5173/index.html`。
2. 说明这是手机版最小闭环：录入加工批次，计算库存和成本。
3. 保持默认数据，点击“生成结果”。
4. 展示结果页：
   - 原料消耗
   - 成品产出
   - 次果和损耗
   - 守恒差异
   - 原料总成本和分摊结果
5. 点击“返回修改”。
6. 把“损耗”改成 `0`，点击“生成结果”。
7. 展示系统会阻断不守恒数据，说明这是为了避免错误成本结果。
8. 最后说明当前边界：本地 demo、无数据库、成本分摊为明确假设。
